1. Project එක මොකක්ද?

Peoples Bakers website එක Sri Lankan bakery business එකක් online වෙන්න build කරපු full-stack web application එකක්. Customer එකට:

  • Products & cakes browse කරන්න
  • Shopping basket එකට items add කර order place කරන්න
  • WhatsApp / phone / PayHere payment use කරන්න
  • Account create කර order history බලන්න
  • Feedback send කරන්න
  • Outlets map එකෙන් branch සොයන්න

Business owner / staff ට Admin panel එකෙන් orders, products, feedback manage කරන්න පුළුවන්.

Important: මේ project එක plain HTML file double-click කරලා open කරන්න බෑ. Node.js server එක run කරලා http://localhost:3000 වලින් open කරන්න ඕන. Live Server, file://, හෝ PHP-only hosting වල backend work වෙන්නේ නැහැ.

2. Frontend vs Backend — simple විස්තරය

Frontend (ඉදිරිපස — user දකින එක)

Browser එකේ run වෙන දේවල්. User click කර, scroll කර, form fill කරන සියල්ල.

  • HTML — page structure (index.html, order.html, …)
  • CSS — colors, layout, mobile design (css/style.css)
  • JavaScript — cart, checkout, language switch, maps (js/ folder)
  • Images — products, hero photos (images/ folder)

Frontend එක alone වලින් orders save වෙන්නේ, login work වෙන්නේ, admin panel live data show වෙන්නේ නැහැ. API call කරන්න backend server එක ඕන.

Backend (පසුපස — server එකේ run වෙන එක)

Computer/server එකේ run වෙන Node.js + Express program එක (server/server.js). මේක:

  • Static files (HTML, CSS, JS, images) browser එකට serve කරනවා
  • REST API endpoints handle කරනවා (/api/...)
  • Login tokens (JWT) verify කරනවා
  • Orders, users, feedback, products SQLite database එකේ save කරනවා
  • PayHere payment webhooks process කරනවා
  • Admin-only data protect කරනවා
┌─────────────┐ HTTP requests ┌──────────────────────┐ │ Browser │ ───────────────────────────► │ Node.js Server │ │ (Frontend) │ ◄─────────────────────────── │ (Backend) │ │ HTML/CSS/JS│ HTML + JSON responses │ server/server.js │ └─────────────┘ └──────────┬───────────┘ │ ▼ ┌──────────────────────┐ │ SQLite Database │ │ peoples-bakers.db │ └──────────────────────┘

Simple analogy: Frontend = shop display window. Backend = kitchen + cashier + record book. Customer window එක බලනවා; order kitchen එකට යනවා database එකේ save වෙනවා.

5. Server run කරන හැටි — Step by Step

Requirements (අවශ්‍ය දේවල්)

  • Node.js 22.5 or newer — nodejs.org download
  • Project folder extract කරලා තියෙන්න ඕන
  • Windows: PowerShell හෝ Command Prompt

Step 1 — Project folder එකට යන්න

cd "C:\Users\...\Peoples-Bakers-master"

Step 2 — Dependencies install (පළමු වතාවට විතර)

npm install

මේක node_modules folder එක create කර dependencies download කරනවා. Internet connection ඕන.

Step 3 — Server start කරන්න

npm start

Terminal එකේ මේ විදිහ message එකක් පෙන්වන්න ඕන:

Peoples Bakers server running at http://localhost:3000

Step 4 — Browser එක open කරන්න

Address bar එකේ type කරන්න:

http://localhost:3000
✓ Success check: Home page load වෙනවා, images පෙන්වනවා, cart button work වෙනවා. Terminal window එක close කරන්න එපා — server stop වෙනවා.

Other useful commands

CommandWhat it does
npm startServer run කරනවා (production mode)
npm run devAuto-restart when files change (development)
npm run verifyBackend health check (server running වෙන්න ඕන)
npm run seedDatabase re-seed (existing data duplicate නොවෙන විදිහට)

Server stop කරන්න

Terminal එකේ Ctrl + C press කරන්න.

15. Production / Hosting (short)

Live internet වල deploy කරන්න:

  • Node.js supported hosting ඕන (cPanel Node, Render, Railway, VPS)
  • Plain PHP-only / GitHub Pages — NOT suitable
  • SSL certificate (HTTPS) — PayHere webhooks සඳහා mandatory
  • Persistent disk for SQLite database
  • Full hosting guide: docs/Peoples-Bakers-Hosting-Guide-Sinhala.pdf
Recommended hosting check: Node.js 22+, 24/7 process, SQLite persistent path, SSL, PayHere notify URL reachable, Singapore DC = OK for SL customers (low latency impact if setup correct).

16. Mobile Responsive — Phone User Experience

Why important? Sri Lanka වල customers බොහෝ දෙනෙක් mobile phone එකෙන් site එක use කරනවා. Website එක phone screen එකට fit වෙන්න, fast load වෙන්න, easy navigate වෙන්න ඕන.

16.1 Responsive design — මොකක්ද?

Responsive කියන්නේ screen size එක (phone, tablet, laptop) අනුව layout automatically adjust වෙනවා. Desktop වල wide grid එක, phone වල single column — images, text, buttons overflow නොවී පෙන්වනවා.

Project එකේ:

  • <meta name="viewport" ...> — phone zoom/layout correct
  • css/style.css — 30+ media queries (768px, 900px, 1024px breakpoints)
  • Grids (products, outlets, footer) → phone වල 1 column
  • Cart side drawer, WhatsApp floating button — touch-friendly
  • PWA manifest — home screen add කරන්න පුළුවන්

16.2 Hamburger menu (☰) — NEW improvement

Problem (before): Phone වල nav links wrap වෙලා navbar එක tall, cluttered වුණා.

Solution (now): Screen width 900px යට:

  • ☰ Hamburger button — tap කර menu open
  • Links side drawer එකක stack වෙනවා (Our History, Products, Order, …)
  • Backdrop tap / link click / Escape key — menu close
  • Order Online button compact (icon only on small screens)

Files: js/main.js (setupMobileNav) + css/style.css (nav drawer styles)

16.3 Slow network / 3G — image speed improvements

Mobile data slow වෙද්දී heavy images site slow කරනවා. Fixes:

FeatureWhat it does
Hero slide deferHome page first hero image only loads fast; other 4 slides load later (staggered)
Hero preloadFirst hero image preloaded for faster first paint
Lazy imagesBelow-fold <img> tags get loading="lazy"
Product card backgroundsHome page product photos load when user scrolls near them

Files: js/main.js (setupProgressiveImages)

16.4 Mobile UX checklist — test කරන්න

  1. Chrome DevTools → Ctrl+Shift+M (mobile view) හෝ real phone
  2. ☰ menu open → link tap → menu close
  3. Home scroll — products images appear when scrolling
  4. Cart → checkout form fill (phone keyboard)
  5. WhatsApp FAB button tap
  6. Language switch සිං on phone
  7. Hard refresh Ctrl+Shift+R after updates
Live hosting වලත්: Mobile responsive CSS/JS server එකෙන් serve වෙනවා — domain live කරපු පස්සේ phone වල same experience. Hosting guide: Peoples-Bakers-Hosting-Guide-Sinhala.pdf (Desktop එකේ).
